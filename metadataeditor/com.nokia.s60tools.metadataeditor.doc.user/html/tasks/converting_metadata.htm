Converting metadata files

By default, new API metadata files are created in version 2.0 format. This format has some differences compared to the earlier 1.0 version syntax.

To convert a version 1.0 metadata file into version 2.0: click the Convert to v.2.0 button.

There are only a few differences between the data formats. In version 2.0:

  • Subsystem is called Collection.
  • For category, SDK and Domain are replaced by Public and Platform, respectively.
  • Extended SDKs are not supported.

The naming pattern of the file itself is still the same. For example, if the API directory is phone_parser_api, the metadata filename is phone_parser_api.metaxml.
